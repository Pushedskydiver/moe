**Novel patterns this session:**

1. **A user pushed back on an `AskUserQuestion` recommendation itself, not just picked between the offered options** — asking whether the recommended fix had real evidence behind it, rather than trusting the finding's own framing. This is the `vision-is-north-star` verify-before-asserting discipline applied one level earlier than before: not just to review findings or design decisions inherited from elsewhere, but to my own recommendation before the user acts on it. Re-verifying against the primary source directly (not the finding's paraphrase) and reporting the exact quotes back before proceeding is now the expected shape whenever this happens again.
2. **The same misquote pattern (a paraphrase in quote-marks presented as verbatim, or an outright-fabricated quote) recurred across six separate locations in one PR**, three files deep, and survived two full review-and-fix cycles before a second R2 pass caught the last instance — in a file (`BUILD_PLAN.md`'s own DA-fold paragraph) added _after_ the first `copilot-surrogate` pass had already run. Fixing found instances doesn't reliably fix the underlying authoring habit within the same PR; a later addition to the same PR can reintroduce the identical defect class the PR's own earlier fixes just closed.
3. **A broken GFM table (a stray blank line splitting a table in two) is a real, `copilot-surrogate`-catchable defect class distinct from column misalignment** — `prettier --check` only reformats/validates what it parses as a table in the first place; a structural break (blank line before the intended next row) doesn't trip it, since Prettier's markdown table formatter treats the fragment as valid prose, not a table cell.
4. **Self-review (the line-level pass, distinct from DA/`copilot-surrogate`'s architectural/doc-layer passes) caught a real staleness independently, after both review agents had already run** — a module TSDoc's own factual claim about which population a type covers, made stale by a same-PR schema change. Confirms self-review is a real backstop, not a formality performed after "the real reviews" already found everything.
5. **A third stacked review-fold arc in one PR** (DA's own fold + R2, then `copilot-surrogate`'s own fold + a second R2) — chunk 3.5 established the dual-fold-arc as expected-not-broken; chunk 3.6 shows a review pass's _own_ fold can itself need a further correction round, not just a confirming pass.
